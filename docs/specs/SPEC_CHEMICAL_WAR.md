# SPEC: Chemical & Deception Warfare (Cortés arc) — CW1…CW3

Governing intent (user, verbatim): "poison bombs should be a thing. of course any poison generated by one
maw will decay. covert pheromone tracks that cause confusion and lead to death or a trap. Hernán Cortés shit
like that." Part of the "rebuild medieval society with intelligent (neural) insectoids" north star; the visible
siege ("catapults and ballistas… siege towers") is the delivery vehicle.

Scope: a maw can now wage CHEMICAL war (a decaying poison cloud, lobbed by a siege engine) and DECEPTION war
(a false pheromone trail that lures an enemy's foragers into that cloud). Baseline-ON, opt-out `--no-poison`,
in `_GATE_NAMES`, byte-identical with the gate off. Reuses the `fires`-dict hazard pattern, the `effects`
projectile pattern, and the per-colony pheromone `trails`.

Cross-references (not duplicated here):
- **SPEC_FAUNA_ECOLOGY FE4** — owns `sim.effects` / `_spawn_effect('shot',…)` / `_effects_tick` (the visible
  travelling projectile). CW2 reuses it for the poison shell's arc.
- **SPEC_TECH TE11** — owns `_catapult_tick`, `CATAPULT_*`, the `catapult` tech, `diplomacy.war_target`.
  CW2 requires the `catapult` (siege) tech as its prereq and mirrors the war-target delivery.
- The `fires`-dict + `_fire_tick` decaying-hazard pattern (per-tick `take_damage`, TTL countdown) is the
  template for the poison field; poison does NOT spread and does NOT require flammable voxels.

## CW1 — The poison cloud (a decaying area hazard)

New transient `sim.poison : Dict[(x,y,z) -> int]` (remaining ticks), getattr/`hasattr`-guarded, plain dict
(checkpoint-safe like `fires`).
- `_seed_poison(center, radius)` — fill every in-bounds cell within Chebyshev `radius` of `center` (at the
  impact z-slice) with `POISON_TTL` ticks (a fresh lob refreshes an existing cell to the max).
- `_poison_tick()` — gated, cadence `POISON_TICK`: every unit within Chebyshev-1 of a poison cell takes
  `POISON_DAMAGE` (resilience-reduced via `unit.take_damage(POISON_DAMAGE, 0.0)`); a kill drops a `CORPSE`.
  Then each poison cell decrements; cells at 0 clear. Deterministic (NO RNG) → byte-identical off.
- Unlike fire, poison ignores voxel type and never spreads — it simply lingers `POISON_TTL` then dissipates
  ("any poison generated by one maw will decay").

**Contract.** Require: none (guarded). Guarantee: `poison` values in `[1, POISON_TTL]`, monotone-decreasing per
cell absent a re-lob; no cell survives past `POISON_TTL` idle ticks; pure (no RNG). Maintain: gate off → the
tick never runs and no `poison` dict is created → battery byte-identical.

**Acceptance CW1.** Seed a cloud over a stationary enemy unit; after `POISON_TICK` steps the unit has taken
damage; after `POISON_TTL` cadence-steps the cloud is empty (`sim._poison() == {}`).

## CW2 — Siege delivery (the poison bomb, visible)

`_poison_bomb_tick()` — gated, cadence `POISON_RELOAD`. A living house that holds the `catapult` (siege) tech
and has a `diplomacy.war_target` within `CATAPULT_RANGE` LOBS a poison shell:
- spawn a visible `_spawn_effect('shot', maw_pos, target_pos)` (reuses FE4 — the shell arcs across the board),
- `_seed_poison(target_maw_pos, POISON_RADIUS)` at the target maw (the cloud blooms on impact).
No direct maw HP damage (poison kills via the cloud over time, not on impact — distinct from the catapult's
instant hit). Deterministic → byte-identical off. This is the second siege engine's payload; the shell is the
"ballista bolt / catapult shot" the user wants to SEE.

**Acceptance CW2.** A catapult-teched house at war with an in-range target: after `POISON_RELOAD` steps an
effect 'shot' is in flight AND a poison cloud exists at the target maw. Gate off → neither appears.

## CW3 — Covert pheromone tracks (the deception; the trap springs) — PHASE B

A saboteur deposits a FALSE `FOOD_TRAIL` into the ENEMY colony's own pheromone channel, leading FROM the
enemy's foragers TOWARD a poison cloud (or a killbox) — the enemy's own chemical guidance betrays them
("covert pheromone tracks that cause confusion and lead to death or a trap"). Gated (same `POISON_ENABLED`
umbrella or its own flag), byte-identical off. Reuses `PheromoneLayer.deposit(pos, enemy_id, FOOD_TRAIL, s)`.
Detailed spec on build. (Deferred to Phase B — poison must exist as the trap first.)

## Constants (sandkings.py)

```
POISON_ENABLED = False   # gate (module default False -> battery byte-identical; entrypoint flips, --no-poison)
POISON_DAMAGE  = 3       # per poison tick, resilience-reduced
POISON_TTL     = 12      # ticks a poison cell lingers before it dissipates (persistent, > fire's 8)
POISON_RADIUS  = 2       # Chebyshev radius of the cloud a shell blooms into
POISON_TICK    = 3       # cadence: poison acts/decays every N steps
POISON_RELOAD  = 60      # steps between poison lobs (slower than the catapult; a special munition)
```

## Acceptance (tests/test_chemical_war.py)

- Gate default OFF; a maxed setup with the gate off is byte-identical (full battery unchanged).
- CW1: cloud damages a unit within it; dissipates after `POISON_TTL`.
- CW2: a war-ready catapult house lobs a visible shell + blooms a cloud at the target on reload.
